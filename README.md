| 技能分类 | 指令/技能名称 | 核心作用 |
| :--- | :--- | :--- |
| 测试 | `test-driven-development` | 强制执行 RED-GREEN-REFACTOR 流程（先写测试，看它失败，写代码，看它通过）。 |
| 调试 | `systematic-debugging` | 4阶段根因分析，拒绝瞎猜，包含防御性编程和条件等待技术。 |
| | `verification-before-completion` | 在宣称修复前，强制验证问题是否真的解决。 |
| 协作/开发 | `brainstorming` | Socratic 设计法。写代码前先提问，细化想法，生成设计文档供你确认。 |
| | `writing-plans` | 将工作拆解为 2-5 分钟能完成的小任务，每个任务包含精确的文件路径和代码。 |
| | `executing-plans` | 批量执行任务，中间设置人工检查点（Checkpoints）。 |
| | `dispatching-parallel-agents` | 并发启动多个子智能体处理不同任务。 |
| | `using-git-worktrees` | 自动创建隔离的分支（Worktree），确保主分支始终干净。 |
| | `finishing-a-development-branch` | 任务完成后，验证测试并询问合并/PR/丢弃的决策。 |
| 元技能 | `writing-skills` | 指导如何编写新的 Superpowers 技能。 |
| | `using-superpowers` | 介绍该技能系统的入门引导。 |
